서버 제출 테스트


  이름과 나이를 입력받아서 서버(java 코드)로 제출

  *준비물 * 

  1) form 태그 : 내부에 작성된 input 값을 지정된 경로로 제출 
  
  - action : 제출할 경로 
  - method : 제출 방법(GET/POST) 기본값: GET 
  
 제출 == 클라이언트가 서버로 요청 

 2)input 태그 값을 입력하는 양식 제공 
  -name (*중요*) : 서버로 제출 시 key 역할 
  
  value or 입력된값 : 서버로 제출시 value 역할 
  



  input 태그 : 값을 입력하는 양식 제공 

 
        k        v
  ?inputName = 홍길동

  **GET : Get 을 통한 요청은 URL 주소끝에 파라미터로 포함되어 전송되며,
  이부분을 query String 이라고 부른다. 
  방식은 url끝에 "?"를 붙이고 그다음 변수명1= 값1 & 변수명2=값2....
  ex) www.example.com/?name1=value1&name2=value2.....
  서버에서는 name1과 name2라는 파라미터 명으로 각각 value 과 value 파라미터 값을 전달받을수 있다. 


  [장점]
  -단순함
  -캐싱 가능 (주소 저장 == 북마크 , 즐겨찾기 , 주소공유 )

  [단점]
  -요청데이터의 길이 제한있다. (너무길면 데이터가 짤림)
  -보안에 취약하다 (요청 데이터가 주소에 그래도 노출됨 )
